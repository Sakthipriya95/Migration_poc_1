Changes of the original source code

We have created a patch org.apache.logging.log4j.patch where we have adapted the changes to LOG4j source code.

The following classes has been copied and modified from Apache Log4j 2.4.0 (log4j-core-2.4.jar).

org.apache.logging.log4j.core.config.AbstractConfiguration - The original source code was modified to supress the message logged in console. The change is commenting Line 430 since we merge all the log4j configurations, so unable to locate appender message is not relevant. The validation for the appenders used is done in finalize method of MergeableConfiguration

org.apache.logging.log4j.core.LoggerContext - The original source code was modified to optimize the initialization performane.Line 131 is commented out and introduced line 132. Now a DefaultConfiguration instance is not created during the static initialization of the class

org.apache.logging.log4j.core.LoggerContext -Line 424 MDGB Build wants to get rid of this warnings. So commented this line. "No Root logger was configured, creating default ERROR-level Root logger with Console appender" The root tag was not present is come log4j xml and hence this warning message was displayed.
